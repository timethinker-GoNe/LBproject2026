🚨 긴급 (보안/안정성)
✅
BinaryFormatter → Newtonsoft.Json 마이그레이션
BinaryFormatter는 deprecated + 보안 취약점 존재. Newtonsoft.Json으로 교체 완료.
기존 바이너리 세이브 파일 자동 변환 지원 (JSON 파싱 실패 → Binary fallback → 다음 저장 시 JSON으로 전환).
📄 FarmingEngine_study/Scripts/Tools/SaveTool.cs
📄 DialogueQuests/Scripts/Tools/SaveTool.cs
📄 Packages/manifest.json (com.unity.nuget.newtonsoft-json: 3.2.1 추가)
⚡ 성능
✅
CraftData.Get(id) Dictionary 캐싱
기존 선형 탐색 O(n) → Dictionary 조회 O(1) 로 최적화.
Load() 시 craft_dict 동시 구축. 아이템 수 증가해도 성능 유지.
📄 FarmingEngine_study/Scripts/Data/CraftData.cs
// Before: O(n) 선형 탐색
foreach (CraftData item in craft_data) {
    if (item.id == id) return item;
}

// After: O(1) Dictionary 조회 ✅
craft_dict.TryGetValue(id, out CraftData result);
return result;
🟡
Plant.cs SlowUpdate → 이벤트 기반 전환
현재 모든 Plant가 0.5초마다 성장 체크를 실행. 농장이 커질수록 연산 증가.
대안: onNewDay 이벤트 구독 + 필요 시에만 성장 계산.
📄 FarmingEngine_study/Scripts/Gameplay/Plant.cs — SlowUpdate()
🟡
TheData.cs 지연 로드 (Lazy Loading)
현재 Awake에서 모든 ScriptableObject를 한번에 로드. 콘텐츠 증가 시 초기 로딩 느려짐.
대안: Addressables 패키지 도입 또는 Resources.LoadAsync 활용.
📄 FarmingEngine_study/Scripts/Data/TheData.cs — Awake()

무엇인가? — UI 패널의 화면 내 위치(앵커)를 Unity 에디터 없이 JSON 파일 하나로 관리하는 시스템입니다. 패널 위치를 바꾸고 싶을 때 프리팹을 열 필요 없이 JSON만 수정하면 됩니다.

1
UIPanel 상속 스크립트 작성
UIPanel을 상속받고 필요한 필드와 Show/Hide override를 작성. 슬롯이 필요하면 ItemSlotPanel 상속.
public class MyPanel : UIPanel {
  [SerializeField] protected string layoutKey = "my_panel";
  public override void Show(bool instant=false) { ... base.Show(instant); }
}
2
Unity에서 Prefab / GameObject 구성
UICanvas 프리팹 또는 씬의 Canvas 하위에 Panel GameObject를 만들고 스크립트를 부착. CanvasGroup이 자동 RequireComponent로 추가됨.
3
UIDesignConfig.json에 레이아웃 등록
StreamingAssets/UIDesignConfig.json에 layoutKey로 앵커 정보 추가. DEV_EDITOR 디자인편집기에서 직접 편집 가능.
{ "my_panel": { "ancMinX": 0.3, "ancMinY": 0.2, "ancMaxX": 0.7, "ancMaxY": 0.8 } }
4
TheUI / PlayerUI에 레퍼런스 연결
전체 차단 패널은 TheUI.IsBlockingPanelOpened()에 추가. 플레이어 인터랙션 패널은 PlayerUI에서 참조.
5
이미지 에셋 연결 방법
배경/테두리 스프라이트는 Image 컴포넌트의 Source Image에 직접 할당. TheUI가 Awake에서 ui_material을 Image에 일괄 적용하므로 Material은 별도 지정 불필요. 새 스프라이트는 UISprite 아틀라스 또는 개별 Sprite로 관리.

🔄 구매 / 판매 흐름
🛒 구매 (BUY)
1. buy_slot 클릭 → 선택
2. BUY 버튼 (gold ≥ buy_cost 시 활성화)
3. SaveData.gold -= buy_cost
4. character.Inventory.GainItem(item, 1)
5. SFX 재생 (buy_sell_audio)
💰 판매 (SELL)
1. 인벤 아이템이 sell_slot 에 자동 표시
2. SELL 버튼 (sell_cost > 0 && CanSell)
3. InventoryData.RemoveItem(item, 1)
4. SaveData.gold += sell_cost
5. SFX 재생 (buy_sell_audio)
CanSell 조건: sell_group == null이면 인벤 전체 판매 허용. GroupData를 지정하면 해당 그룹에 속한 아이템만 판매 가능.
