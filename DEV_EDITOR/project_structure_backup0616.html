🚨 긴급 (보안/안정성)
✅
BinaryFormatter → Newtonsoft.Json 마이그레이션
BinaryFormatter는 deprecated + 보안 취약점 존재. Newtonsoft.Json으로 교체 완료.
기존 바이너리 세이브 파일 자동 변환 지원 (JSON 파싱 실패 → Binary fallback → 다음 저장 시 JSON으로 전환).
📄 FarmingEngine_study/Scripts/Tools/SaveTool.cs
📄 DialogueQuests/Scripts/Tools/SaveTool.cs
📄 Packages/manifest.json (com.unity.nuget.newtonsoft-json: 3.2.1 추가)
⚡ 성능
✅
CraftData.Get(id) Dictionary 캐싱
기존 선형 탐색 O(n) → Dictionary 조회 O(1) 로 최적화.
Load() 시 craft_dict 동시 구축. 아이템 수 증가해도 성능 유지.
📄 FarmingEngine_study/Scripts/Data/CraftData.cs
// Before: O(n) 선형 탐색
foreach (CraftData item in craft_data) {
    if (item.id == id) return item;
}

// After: O(1) Dictionary 조회 ✅
craft_dict.TryGetValue(id, out CraftData result);
return result;
🟡
Plant.cs SlowUpdate → 이벤트 기반 전환
현재 모든 Plant가 0.5초마다 성장 체크를 실행. 농장이 커질수록 연산 증가.
대안: onNewDay 이벤트 구독 + 필요 시에만 성장 계산.
📄 FarmingEngine_study/Scripts/Gameplay/Plant.cs — SlowUpdate()
🟡
TheData.cs 지연 로드 (Lazy Loading)
현재 Awake에서 모든 ScriptableObject를 한번에 로드. 콘텐츠 증가 시 초기 로딩 느려짐.
대안: Addressables 패키지 도입 또는 Resources.LoadAsync 활용.
📄 FarmingEngine_study/Scripts/Data/TheData.cs — Awake()

1
UIPanel 상속 스크립트 작성
UIPanel을 상속받고 필요한 필드와 Show/Hide override를 작성. 슬롯이 필요하면 ItemSlotPanel 상속.
public class MyPanel : UIPanel {
  [SerializeField] protected string layoutKey = "my_panel";
  public override void Show(bool instant=false) { ... base.Show(instant); }
}
2
Unity에서 Prefab / GameObject 구성
UICanvas 프리팹 또는 씬의 Canvas 하위에 Panel GameObject를 만들고 스크립트를 부착. CanvasGroup이 자동 RequireComponent로 추가됨.
3
UIDesignConfig.json에 레이아웃 등록
StreamingAssets/UIDesignConfig.json에 layoutKey로 앵커 정보 추가. DEV_EDITOR 디자인편집기에서 직접 편집 가능.
{ "my_panel": { "ancMinX": 0.3, "ancMinY": 0.2, "ancMaxX": 0.7, "ancMaxY": 0.8 } }
4
TheUI / PlayerUI에 레퍼런스 연결
전체 차단 패널은 TheUI.IsBlockingPanelOpened()에 추가. 플레이어 인터랙션 패널은 PlayerUI에서 참조.
5
이미지 에셋 연결 방법
배경/테두리 스프라이트는 Image 컴포넌트의 Source Image에 직접 할당. TheUI가 Awake에서 ui_material을 Image에 일괄 적용하므로 Material은 별도 지정 불필요. 새 스프라이트는 UISprite 아틀라스 또는 개별 Sprite로 관리.

story_events.json · npc_idle.json을 런타임에 로드해 게임 이벤트 트리거와 NPC 대사를 처리. QuestManager의 진행 상태를 구독해 조건 충족 시 이벤트 분기.
EventSystemUnity Built-in
인게임 내 모든 UI 이벤트(인벤토리 클릭, 퀘스트 패널 버튼, 설정 슬라이더 등) 처리. 씬에 하나만 있어야 함.
📷 CAMERA
Main Camera
3D 게임 월드를 렌더링하는 메인 카메라. FarmingEngine의 TheCamera.cs가 플레이어 추적, 줌, 화면 셰이크를 제어. UI 씬의 UICamera와 달리 월드 공간을 렌더링.
🖼️ UI — SaveUI_Canvas
SaveSlotPanelFarmingEngine
인게임 저장/로드 UI. SlotUI_0 · SlotUI_1 · SlotUI_2 세 개 슬롯 표시. 각 슬롯에 날짜·플레이 시간 표시. SaveManager와 연동해 저장 실행 및 슬롯 선택 로드 처리.
🖼️ UI — QuestUI
FarmingQuestPanelFarmingEngine · J키
J키로 여닫는 전체 퀘스트 저널. 카드 형식으로 퀘스트 목록·목표·보상을 표시. QuestManager에서 상태를 받아 진행중·완료·미수락 탭으로 분류.
Tracker미니 트래커
화면 우상단 상시 표시 미니 퀘스트 트래커. 현재 활성 퀘스트의 목표 항목과 진행 수치를 간략하게 표시. 퀘스트 없을 때 자동 숨김.
Toast알림 팝업
퀘스트 수락·목표 달성·완료 시 화면에 잠깐 표시되는 팝업 알림. QuestNotificationBridge 대체. QuestManager 이벤트 구독으로 자동 트리거.
🖼️ UI — 공통 패널
SettingsPanelFarmingEngine
인게임 설정 패널. 시작화면과 동일한 Tab_Audio · Tab_Display · Tab_Keys 구조. ESC 또는 메뉴 버튼으로 열림. 설정값은 PlayerPrefs에 저장되어 씬 간 유지.
FullInventoryPanelFarmingEngine
전체 아이템창(커스텀 인벤토리). 수집한 아이템 전체를 카테고리별로 표시. FarmingEngine의 Inventory(퀵슬롯)와 별개로 동작하는 커스텀 UI 레이어.
ConfirmPanelFarmingEngine
인게임 내 위험 작업(저장 덮어쓰기, 아이템 폐기 등) 전 이중 확인 다이얼로그.
MessageText · DayText · PlayTimeText
HUD 텍스트 요소. MessageText는 시스템 메시지 표시, DayText는 게임 내 날짜, PlayTimeText는 누적 플레이 시간. TheGame에서 매 프레임 또는 이벤트 시 업데이트.
🌍 WORLD
Ground
씬의 기본 지형 메시. 플레이어와 오브젝트가 위에 배치되는 바닥면. 충돌체(Collider) 포함.
SoilArea
농사 가능 구역. 플레이어가 호미(Hoe)로 밭을 일구고 씨앗을 심을 수 있는 영역. FarmingEngine의 그리드 시스템과 연동.
👤 CHARACTERS
Actor 오브젝트들DialogueQuests.Actor
씬에 배치된 NPC 및 상호작용 가능 캐릭터. DialogueQuests의 Actor 컴포넌트를 통해 대화·퀘스트 부여 가능. StoryEventManager의 npc_idle.json으로 유휴 대사 관리.
🌤️ ENVIRONMENT · QUESTS
퀘스트 존 마커[Quests] 그룹
퀘스트 목표 위치를 지정하는 트리거 존. QuestManager의 목표 조건(특정 구역 도달, 오브젝트 상호작용 등)과 연결. 씬 계층에서 [Quests] 그룹 아래 배치.
Environment 오브젝트
배경 나무·풀·울타리·건물 등 시각적 환경 요소. 대부분 정적(Static) 오브젝트로 배치돼 GPU 인스턴싱 대상. [Environment] 그룹 아래 정리.
