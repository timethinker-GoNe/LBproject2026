# 경영 시스템 전략 (베이커리 판매)

작성: 2026-07-04
목적: 베이커리에서 빵을 진열하면 손님이 찾아와 구매하는 "경영 시스템" 설계.
상태: **설계 완료, 구현 시작 전 — 다음 세션에 아이디어 추가 후 이어서 진행**

## 배경

- `BakeryOven`/`BakingPanel` = 빵 제작 (완료)
- `BreadDisplayShelf`/`DisplayShelfPanel` = 진열 (완료, 단 현재는 순수 `StoragePanel` 재사용이라 진열해도 판매 로직 없음 — 그냥 저장고)
- 오늘 다룰 것: 진열된 빵을 손님 NPC가 실제로 사가서 골드를 버는 "판매 루프"
- 관련 기존 메모리: `bakery_system_strategy.md`(제작/진열 구현 상세), `project_shop_system.md`("경영 시스템 탭 기획" 항목)

## 확정된 설계 결정 (2026-07-04 세션)

| 항목 | 결정 | 이유/비고 |
|---|---|---|
| 손님 표현 | **실제 손님 NPC** (이동 + 구매 애니메이션) | 추상 판매(타이머 틱)보다 개발량 크지만 몰입감 위해 선택. 신선도 시스템은 생략했으니 그만큼 여유로 감당 |

## 그라운드 확인된 재사용 가능 아키텍처 (코드 확인 완료, 2026-07-04)

- **`Character.cs`** (`Scripts/Gameplay/Character.cs`) — `MoveTo(Vector3)`, `StopMove()`, `HasReachedMoveTarget()` 보유. NavMesh 경로 계산은 `NavMeshTool.CalculatePath`가 내부적으로 처리.
- **`AnimalWild.cs`** 패턴 — `[RequireComponent(typeof(Character))]`로 컴포지션, `character = GetComponent<Character>()` 후 이동 API 호출. enum state + `Update()` switch 상태머신 구조. 손님 AI에 그대로 적용할 템플릿.
- **`Spawner.cs`** (`Scripts/Gameplay/Spawner.cs`) — 주기적 스폰 타이머 패턴. `TheGame.Get().GetGameTimeSpeedPerSec()` 기반 타이머 증가, `PlayerData.Get().SetCustomFloat(uid, timer)`로 세이브 지속. `spawn_max`/반경 체크로 동시 개체 수 제한.
- **`TheGame.cs`** — `onNewDay` 이벤트(라인 20), `pdata.day_time`(0~24f), `gdata.start_day_time`/`end_day_time` (기존 `IsNight()`류 판단에 이미 쓰임, 라인 318 근방).
- **`ItemData.cs`** — `buy_cost`/`sell_cost` 필드가 이미 `ShopPanel`에서 쓰이는 가격 소스. `[Header("Shop")]` 아래 `sell_able` 필드가 최근 추가된 전례 있음 (같은 자리에 `customer_demand` 추가 예정).
- **`PlayerCharacterData.cs`** (`Scripts/Data/PlayerCharacterData.cs:37`) — `public int gold`. `PlayerCharacter.Get(0).SaveData.gold`로 백그라운드에서도 접근 가능 (플레이어 상호작용 컨텍스트 불필요, `PlayerCharacter.Get(int player_id=0)` 정적 접근자 확인함).
- **`InventoryData.Get(InventoryType.Storage, uid)`** — `BreadDisplayShelf.RefreshDisplayItems()`가 이미 이 방식으로 진열대 재고를 읽고 있음.
