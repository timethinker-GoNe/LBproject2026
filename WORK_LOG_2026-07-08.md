## 4. 패널을 닫아도 베이킹 시간이 흐르도록 수정

요구사항:

- 베이킹 패널을 닫아도 굽는 시간은 계속 흘러야 합니다.
- 단순히 패널이 열려 있을 때만 진행되는 구조는 맞지 않습니다.

수정 파일:

- `Assets/FarmingEngine_study/Scripts/UI/BakingPanel.cs`

수정 내용:

- 기존에는 `Update()`에서 `IsVisible()` 체크 이후에 베이킹 타이머가 처리되었습니다.
- 이 구조에서는 패널을 닫으면 타이머가 진행되지 않았습니다.
- `UpdateBakingTimer()`를 분리하고, `IsVisible()` 체크보다 먼저 호출되도록 변경했습니다.
- 패널이 닫혀 있어도 `BakingPanel` 컴포넌트가 활성 상태라면 시간이 계속 흐릅니다.
- 패널을 다시 열었을 때 진행 중/완료 상태가 UI에 다시 반영되도록 `RefreshBakingStateUI()`를 추가했습니다.
- 완료 후 패널을 다시 열어도 결과물 수령 UI가 유지되도록 처리했습니다.
- 결과물을 수령하면 `pending_result`, `bake_timer`, 진행바, 상태 텍스트를 정리하도록 수정했습니다.

현재 동작 기준:

- `Hide()`로 패널만 닫는 경우: 베이킹 시간 계속 진행
- 패널을 다시 여는 경우: 진행 중이면 남은 시간 표시, 완료되었으면 결과물 수령 표시
- `UIPanel.Hide()`가 fade-out 이후 GameObject를 비활성화하던 문제가 있어, `BakingPanel.AfterHide()`를 오버라이드해서 닫힌 상태에서도 GameObject가 유지되도록 수정
- `bake_end_time`을 추가해서 `Time.time` 기준으로 완료 시간을 계산하도록 보강
- 패널이 닫혀 `Update()` 호출이 잠시 끊기는 상황이 생겨도, 다시 열 때 실제 경과 시간이 반영되도록 수정

## 5. `Scene_Farm_01`의 중복 `BakingPanel` 정리

문제:

- `Scene_Farm_01` 안에 `BakingPanel`이 2개 존재했습니다.
- `BakingPanel`은 static `_instance`를 사용하는 구조라서, 중복 패널이 있으면 어떤 패널이 실제로 사용될지 Awake 순서에 의존할 수 있었습니다.
- 버튼 이벤트, 레시피 목록, 결과물 UI가 서로 다른 패널을 참조할 위험이 있었습니다.

수정 파일:

- `Assets/Scenes/Game/Scene_Farm_01.unity`
- `Assets/FarmingEngine_study/Scripts/UI/BakingPanel.cs`

수정 내용:

- `DQUICanvas` 쪽에 추가되어 있던 중복 `BakingPanel` 서브트리를 제거했습니다.
- `UICanvas` 쪽 `BakingPanel`만 남겼습니다.
- `BakingPanel.Awake()`에 중복 인스턴스 방어 코드를 추가했습니다.
- 이미 `_instance`가 존재하는 상태에서 다른 `BakingPanel`이 깨어나면 경고 로그를 남기고 해당 중복 패널을 비활성화합니다.

현재 확인 결과:

- `Scene_Farm_01.unity`에서 `BakingPanel`은 1개만 검색됩니다.
